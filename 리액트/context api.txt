###### context api ######

- A 컴포넌트에서 props를 통해 넘겨받은 함수를 A 에서는 핋요가 없지만 A의 자식 컴포넌트인
B 컴포넌트에서만 해당 함수를 필요로 하는 경우가 있다. 이 경우에 사실 A에서는 함수를 props를 
통해 전달해주기만 하는 역할을 하고 있다. 이렇게 한 두개의 컴포넌트를 거쳐서 전달하는
것은 큰 불편함은 없지만 만약 3 ~ 4개 이상의 컴포넌트를 거쳐서 전달해하는 경우에는 꽤나 번거로워
질 것이다. 이 때 Context api와 dispatch를 함께 사용하면 큰 도움이 된다.
- context api를 사용하면 프로젝트안에서 전역적으로 값을 관리할 수 있다. 
    --여기서 '상태'가 아니라 '값'이다. 즉 이 값은 꼭 상태를 가르키지 않아도 된다. 이 값은
    함수일수도 있고 어떤 외부 라이브러리 인스턴스일수도 있고 심지어 DOM일수도 있다.
