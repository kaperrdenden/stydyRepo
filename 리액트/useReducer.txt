###### useReducer ######

- state를 관리할 때 useState외에도 방법들이 존재하는데 그 중에 하나가 useReducer이다.
이 hook 함수를 사용하면 컴포넌트의 state 업데이트 로직을 컴포넌트에서 분리시킬 수가 있다.
로직을 컴포넌트 바깥에 작성할 수도 있고 심지어는 다른 파일에 작성 후 불러와서 사용하는 방법도
가능해진다.
- useReducer은 현재 상태(state)와 '액션 객체'를 파라미터로 받아와서 새로운 상태를 반환해주는
함수이다. 
- reducer 함수에서 반환하는 state는 곧 컴포넌트가 지닐 새로운 state가 된다. 

- 기본적인 형태는 const [<상태 객체>, <dispatch 함수>] = useReducer(<reducer 함수>, <초기 상태>, <초기 함수>)
이와 같다. 
    - redecer 함수는 현재상태(state)객체와 행동(action) 객체를 인자로 받아서 새로운
    state 객체를 반환하는 함수이다. (즉 reducer 함수를 통해서 state를 변화시킴)
    - dispatch함수는 컴포넌트 내에서 상태 변경을 일으키기 위해서 사용되는데 인자로 
    redecer함수에 넘길 action 객체를 받는다. action 객체는 관행적으로 어떤 부류의 action인지를
    나타내는 type 속성과 해당 action과 관련된 데이터를 담고 있다.
    다시 말해 dispatch 함수에 action을 던지면 reducer 함수가 이 action에 따라 
    state를 변경해준다.
